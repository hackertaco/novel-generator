2. 하네스 (NovelHarness)

설정 기반 실행 엔진

파이프라인, 모델, 평가 기준, 트래킹을 조합 가능한 설정으로 분리. 같은 시드로 다른 프리셋을 돌려 비교 가능.

프리셋 모델 파이프라인 임계값 빠른 모드 호출 수/화
기본 gpt-5.4 Writer → Guard → Constraint → Consistency → Quality → Polisher 0.85 ❌ (비트별) ~25-35
절약 gpt-4o Writer → Guard → Constraint → Consistency → Quality → Polisher 0.75 ❌ ~20-30
빠름 gpt-4o-mini Writer → Guard → Constraint 0.70 ✅ (씬 단위) ~3-5

정보이론 기반 서사 구조 분석 (21%)

Shannon Entropy, Jensen-Shannon Divergence, Pearson 상관계수 — 학술 논문 검증 방법론을 시드/블루프린트 시스템에 적용

엔트로피 역동성 (20%)
문단별 감정 분포의 Shannon 엔트로피 → 표준편차
높음 = 감정 변화 풍부 (역동적)
낮음 = 감정 단조로움 (지루)
피봇 실현 (35%)
텍스트 1/3, 2/3 지점에서 JSD 측정
JSD > 0.1 = 반전/전환이 실현됨
JSD < 0.1 = 의도된 반전이 묻힘
아크 상관계수 (30%)
블루프린트 tension_level 곡선 vs 실제 감정 곡선
Pearson r > 0.6 = 의도대로 작성됨
Pearson r < 0.3 = 의도와 괴리
정체 방지 (15%)
JSD < 0.05인 문단이 3개 이상 연속 = 정체 구간
0개 = 완벽
3+ 연속 = "아무 일도 안 일어남" 경고
학술적 근거
• Narrative Information Theory (arXiv:2411.12907) — JSD로 서프라이즈/피봇 감지
• Reagan et al. "Six Basic Shapes" — 5만 소설 분석, 감정 곡선 형태 분류
• Fabula Entropy Indexing (arXiv:2104.07472) — 엔트로피 기반 코히어런스 측정

정보이론 적용 전/후 비교

감지 대상Before (프록시)After (정보이론)

반전 효과 부정어 증가 여부 JSD 스파이크 + 블루프린트 tension 대조

11. 한계 인식

✅ 결정적 평가가 잘 하는 것

  • 기본적인 실수 잡기 (어미 반복, 대사 부족, 분량 초과)
  • "최소 품질 기준선" 설정 (이 이하면 확실히 나쁨)
  • 빠른 1차 필터링 (LLM 호출 전에 명백한 문제 제거)
  • 논리적 모순 감지 (죽은 캐릭터, 시간선 역설)
  • 정체 구간 자동 감지 — JSD로 "아무 일도 안 일어나는" 구간 특정
  • 묻힌 반전 감지 — 블루프린트 의도 vs 실제 JSD 비교
  • 의도 이행률 — 감정 곡선 상관계수로 "얼마나 의도대로 썼는가?"

❌ 여전히 LLM이 필요한 것

  • "이 글이 감동적인가?" — 감정 사전은 키워드만, 맥락은 모름
  • "이 반전이 놀라운가?" — JSD는 "변화가 있었다"만, "좋은 변화인지"는 모름
  • "이 대사가 자연스러운가?" — 패턴 매칭은 형태만, 뉘앙스는 모름
  • 독자의 실제 반응 예측 — 구조적 신호일 뿐, 종합적 재미 판단은 불가
결론: 정보이론 도입으로 "확실히 나쁜 것"에서 "구조적으로 문제인 것"까지 범위 확장.
"진짜 좋은 것"을 판단하는 건 여전히 LLM의 몫. 둘을 조합하면 비용 70% 절감 + 품질 유지.
